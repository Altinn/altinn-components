# Profile

Profile pages for user settings and account management. It should be accessible from the main menu.

## Overview

The starting page should give an overview of the user's settings. (Work-in-progress.)

<Canvas of={ProfileStories.ProfilePage} />

## Accounts

Overview of available accounts seen from the end user perspective. The page should only available for users with multiple actors. Expanding an account should shows more details, grouped into relevant parts, like roles, access management, notification settings per account.

<Canvas of={ProfileStories.AccountsPage} />

## Alerts

Notification settings and notification settings per actor. Settings per actor are opened in i modal. The same setting can be accessed via the Accounts page.

<Canvas of={ProfileStories.AlertsPage} />

## Bookmarks

Overview of bookmarks collected på the end user. For now this will be saved searches from inbox, but can be expanded to include other types of bookmarks, like frequently used forms that the user wants access to.

<Canvas of={ProfileStories.BookmarksPage} />

## Activity log

The activity log should include a history of actions performed by the user, across different services and domains.
